السؤال 1 تحديد القيم المجهولة a, b, c, x, y, z

1

المبدأ: حساب المتوفر من المجموع والتخصيص

الموارد إمّا مخصّصة للعمليات أو متوفّرة، لذلك:

Total = Disponible + Σ(Allocation)  ⟹  Disponible = Total − Σ(Allocation)

نحسب مجموع أعمدة مصفوفة Allocation حسب النوع:

Σ A = 1 + x + 3 + 1 = 5 + x Σ B = 1 + 0 + y + 2 = 3 + y Σ C = 0 + 1 + 2 + 1 = 4

بتعويضها في علاقة المتوفّر:

a = 12 − (5 + x) = 7 − x b = 8 − (3 + y) = 5 − y c = 7 − 4 = 3 ✓ مباشرة z = 2 ✓ معطاة

إذًا c = 3 و z = 2 مُحدّدان تمامًا. يبقى إيجاد x و y من العلاقة x + y = 5.

2

كل الحلول الممكنة لـ x + y = 5

المعادلة x + y = 5 لا تعطي حلاً وحيدًا. بما أنّ x, y أعداد طبيعية (عدد وحدات مخصّصة) ويجب أن يبقى Disponible ≥ 0، نحصل على ستّة مُرشّحين. لكل مُرشّح نحسب a = 7−x و b = 5−y و c = 3:

  • P1 و P4 تُنفّذان مباشرة دائمًا (حاجتهما من B تساوي 0).
  • P3 تُنفّذ مباشرة إذا كان b ≥ 1 أي y ≤ 4.
  • P2 تُنفّذ مباشرة إذا كان b ≥ 2 أي y ≤ 3.

لذلك لكي تُنفّذ العمليات الأربع مباشرة نحتاج y ≤ 3، أي المُرشّحون x = 2, 3, 4, 5.

✅
حلول مكافئة تمامًا للأسئلة 3 و 4

المُرشّحات (x=2,y=3)، (x=3,y=2)، (x=4,y=1)، (x=5,y=0) كلّها تُعطي نظامًا آمنًا وتسمح بتنفيذ العمليات الأربع مباشرة. الفرق الوحيد بينها هو توزيع التخصيص حسب النوع (السؤال 2)، أمّا المجموع الكلي فيبقى 17 وحدة دائمًا.

⚠️
لماذا لا نختار x=0 أو x=1؟

هذان الحلّان يُبقيان النظام آمنًا (السؤال 3 له حل عبر سلسلة مثل <P1,P4,...>)، لكنّهما لا يسمحان بتنفيذ كل العمليات مباشرة: عند x=0 تُحجب P2 و P3 (لأنّ b=0)، وعند x=1 تُحجب P2 (لأنّ b=1<2). لذلك لا يطابقان النتيجة المعيارية للسؤال 4 («الأربع تُنفّذ مباشرة»).

نختار المُرشّح x = 2, y = 3 لأنّه القيمة المعتمدة في التصحيح: يُعطي Disponible = (5, 2, 3) وتوزيع تخصيص (7, 6, 4)، ويحقّق كل الأسئلة باتساق.

السؤال 5 أصغر تعديل على Allocation يسبّب انسدادًا كليًا

كلما زاد التخصيص صغُرت قيمة Disponible. الانسداد الكلي (Interblocage Total) يعني أنّ كلّ عملية محجوبة: لكل عملية نوع مورد فيه Besoin > Disponible.

# شروط حجب كل عملية P1 : dispoA < 1 أو dispoC < 1 P2 : dispoA < 2 أو dispoB < 2 P3 : dispoB < 1 أو dispoC < 1 P4 : dispoA < 1 أو dispoC < 2

أرخص حل: نُنقص C بـ 3 (من 3 إلى 0) و B بـ 1 (من 2 إلى 1) عبر زيادة التخصيص بنفس القدر، فيصبح Disponible = (5, 1, 0):

Disponible = (5, 1, 0) P1 : Besoin C = 1 > 0 → محجوبة ⛔ P2 : Besoin B = 2 > 1 → محجوبة ⛔ P3 : Besoin C = 1 > 0 → محجوبة ⛔ P4 : Besoin C = 2 > 0 → محجوبة ⛔
⛔
أصغر تعديل = إضافة 4 وحدات للتخصيص

ΔAlloc = (0, +1, +3) ➜ Disponible = (5, 1, 0) ➜ Interblocage Total. بأقل من 4 مستحيل: حجب P1, P3, P4 يفرض تنقيص C بـ 3 وحدات، وحجب P2 يفرض وحدة إضافية على A أو B.

التمرين 3 — الساعات الشعاعية (Horloges Vectorielles)

نص السؤال

في نظام موزّع يستعمل الساعات الشعاعية. الطوبولوجيا رسم بياني كامل (Graphe complet)، عدد المحطات 3: P0, P1, P2.

  • يوجد حدث عند كل لحظة t = 2, 4, 6, 8, 10, …
  • المحطة ذات الهوية (t % 3) تُرسل إلى المحطة ((t+1) % 3) الرسالة Mₖ,ₗ
  • حيث k = رقم الرسالة للمرسل، و l = t % 3 (المحطة المُرسِلة).

الأسئلة:

  1. ارسم مخطّط الزمن (Diagramme de temps) إلى غاية اللحظة 20.
  2. أعط قيمة شعاع الساعة (Vecteur d'horloge) لكل محطة.
  3. إذا كان زمن نقل كل رسالة على كل قناة هو 1 ثانية، أعط تفاصيل الرسالة المرسلة عند اللحظات t = 12, 28, 50, 70.
  4. إذا أراد كل من P0 و P2 الدخول إلى <SC> عند t=8 و t=12، فماذا يصبح الرسم حسب خوارزمية Agrawala، مع <SC> = 1s.

📖 الفكرة النظرية: الساعة الشعاعية

في الأنظمة الموزّعة لا توجد ساعة عالمية واحدة. الساعة الشعاعية (Vector Clock) تسمح بتحديد الترتيب السببي (causalité) بين الأحداث. كل محطة Pi تحفظ شعاعًا Vᵢ = [Vᵢ[0], Vᵢ[1], Vᵢ[2]]، حيث Vᵢ[j] = عدد أحداث Pj التي تعرفها Pi.

قواعد التحديث (نبدأ بـ [0,0,0]):

  • حدث داخلي أو إرسال: Vᵢ[i] += 1 (ثم تُرسل نسخة من Vᵢ مع الرسالة).
  • استقبال رسالة تحمل W: أولاً Vᵢ[i] += 1، ثم لكل j: Vᵢ[j] = max(Vᵢ[j], W[j]).

الجدولة الدورية للإرسال (دورة 6): عند t=2 ترسل P2→P0، t=4 ترسل P1→P2، t=6 ترسل P0→P1، ثم يتكرر النمط. مع زمن نقل = 1ث: الرسالة المرسلة في t تصل في t+1.

🧩 السؤال 4 — دخول P0 و P2 إلى المنطقة الحرجة حسب Ricart–Agrawala

خوارزمية Ricart–Agrawala للإقصاء المتبادل الموزّع: عندما تريد محطة دخول <SC> تُرسل إلى كل المحطات رسالة REQUEST(timestamp, Pi). كل محطة تستقبل الطلب إمّا تردّ فورًا بـ OK/REPLY، أو تؤجّل الردّ إذا كانت هي أيضًا طالبة للدخول ولها أولوية أكبر. تدخل المحطة <SC> فقط بعد استلام OK من جميع المحطات الأخرى.

قاعدة الأولوية: الطابع الزمني الأصغر أولًا، وعند التساوي نختار رقم المحطة الأصغر. هنا نقرأ نص السؤال بالترتيب المذكور: P0 يطلب عند t=8 و P2 يطلب عند t=12، ومدة البقاء في <SC> هي 1s.

عند t = 8: المحطة P0 تطلب الدخول، إذن ترسل REQUEST(8, P0) إلى P1 و P2.
لا توجد محطة أخرى طلبت <SC> بطابع أصغر في هذه اللحظة، لذلك P1 و P2 يرسلان OK إلى P0.
بعد استلام P0 لكل الردود OK تدخل P0 إلى <SC> أولًا لمدة 1s، ثم تخرج وتحرّر القسم الحرج.
إذا اعتبرنا زمن نقل رسائل REQUEST/OK = 1s، فإن REQUEST تصل عند t=9، و OK يعود عند t=10، فتكون فترة دخول P0 تقريبًا من t=10 إلى t=11. وإذا أهملنا زمن نقل رسائل Agrawala، فالترتيب يبقى نفسه: P0 قبل P2.
عند t = 12: المحطة P2 تطلب الدخول، فترسل REQUEST(12, P2) إلى P0 و P1.
في هذه اللحظة تكون P0 قد خرجت من <SC> لأن مدة القسم الحرج 1s فقط، لذلك لا يوجد تعارض فعلي مع طلب P2.
P0 و P1 يرسلان OK إلى P2، وبعد استلام الردود تدخل P2 إلى <SC> ثانيًا لمدة 1s.
حتى لو حدث تداخل نظريًا، فالطلب ذو الطابع الأصغر هو REQUEST(8, P0)، لذلك الأولوية تبقى لـ P0 ثم P2.

سنبدأ من المخطط الأصلي للامتحان، نُعيد بناء الرسائل والأحداث الناقصة بالتبرير، ثم نحسب مصفوفة الساعة لكل حدث على حدة باستعمال خوارزمية Raynal الرسمية — حتى تتمكّن من حلّ أي تمرين مشابه بنفسك.

نص التمرين المعطيات الأصلية

لدينا نظام موزّع مكوّن من أربعة مسارات (Processus): P1, P2, P3, P4. تتبادل هذه المسارات رسائل فيما بينها، وكل مسار يحتفظ بمصفوفة ساعة Matrix Clock من الحجم 4×4.

المخطط الزمني (Chronogramme) المعطى في الامتحان غير مكتمل: بعض الرسائل لم تُرسم، لكنّ الامتحان يعطينا قيمة مصفوفة الساعة عند الحدث e1 (وهو حدث الإرسال على P1):

الهدف من التمرين

أن نستعمل المعلومة الوحيدة المؤكَّدة (مصفوفة e1) كـ«دليل» لاكتشاف ما هو ناقص في الرسم، ثم نُعيد حساب كل المصفوفات من الصفر للتأكّد من صحّة المخطط واحترامه لشرط التسليم (Condition de délivrance).

تذكير سريع بقواعد Raynal Matrix Clock

كل مسار Pi يملك مصفوفة M مبدئيًا أصفار. الصف i يمثّل ما يعرفه Pi عن إرسالاته، وباقي الصفوف ما يعرفه عن غيره.

  • حدث محلي: M[i][i] += 1 فقط.
  • إرسال (Send) إلى Pj: M[i][i] += 1 ثم M[i][j] += 1، ثم تُرفق نسخة من M مع الرسالة.
  • استقبال (Receive) من Pj لرسالة تحمل W: M[i][i] += 1 ثم M[x][y] = max(M[x][y], W[x][y]) لكل خانة.
  • شرط التسليم عند استقبال Pi رسالةً من Pj تحمل W: يجب أن يكون W[j][i] = M[j][i] + 1 و W[k][i] ≤ M[k][i] لكل k ≠ j.
